GIS では座標を扱うが、それに関する基礎知識をまとめる。

> [!NOTE]
> 基本的な部分のみの記述であることに留意。

## 緯度と経度

GIS においては大抵の場合、位置を**緯度 (latitude)** と**経度 (longitude)** の座標によって表す。

### 緯度と経度の概要

- **緯度 (latitude)**
  - 緯度は地球での**南北**の位置を表す。
  - 地球の中心から見て、赤道を0度として北と南に何度ずれているか。
  - 北緯・南緯。
  - 北へ向かうと北緯が増えていき、北極点で北緯90度となる。逆に南極点では南緯90度になる。
  - 基本は北緯・南緯それぞれ `0〜90`度 で表すが、北緯を基準として `-90〜0〜90`度 の数値のみで表すこともある。
  - 緯度は気候への影響が大きい。およそ、「緯度の数字が大きい（高緯度） = 寒冷地である」と言える。
- **経度 (longitude)**
  - 経度は地球での**東西**の位置を表す。
  - 地球の中心から見て、イギリス・ロンドンの旧グリニッジ天文台（の跡地のあたり）を0度として、東と西に何度ずれているか。
  - 東経・西経。
  - 東へ向かうと東経が増えていき、地球の反対側の（ほぼ）太平洋上にある日付変更線で東経180度となる。逆周りだと西経180度になる。
  - 基本は東経・西経それぞれ `0〜180`度 で表すが、東経を基準として `-180〜0〜180`度 の数値のみで表すこともある。
  - 経度は時間帯（タイムゾーン）に影響する。経度15度で1時間の時差が生じる。
    - `360度 / 24時間` の計算による単純な数値。実際の時差は国などの境界で決まる。

しかしながら、緯度と経度は地球という球体での位置を表現するものであるため、実際に GIS で扱う場合にはいくつか留意点がある。

### 緯度と経度だけでは実際の位置が分からない

緯度と経度だけで分かるのは球面上（= 地表面）での位置だけであり、高度（標高）が分からないからである。

海面に近い平地にいるのか、山の上にいるのか、空を飛んでいるのか、はたまた地面に潜っているのかで、同じ緯度経度であっても実際の位置は異なってくる。これを含めて扱うには**高度 (altitude)** を導入する必要がある。

ちなみに緯度経度は **度** を単位とすることが多いが、高度は **メートル** が単位になることが多い。

### 経度1度の実際の長さは、緯度によって変わる

緯度1度の長さは地球上のどこでもあまり変わらない（おおむね 111km 前後）が、経度1度の実際の長さは緯度によって極端に変わる。

これは、経度の線が北極点と南極点を通るためであり、赤道では長く（緯度と同じく約111km）、極端に言えば北極点と南極点では0kmとなる。

したがって、完全な南北移動以外の距離を測る場合は、地球の形状（完全な球体ではなく南北に少し潰れた回転楕円体）の関係もあり、複雑な計算を行う必要がある。これについては英語ではあるが [Wikipedia の Geographical distance](https://en.wikipedia.org/wiki/Geographical_distance) が参考になる。概要は[日本語版の Wikipedia](https://ja.wikipedia.org/wiki/%E7%B5%8C%E7%B7%AF%E5%BA%A6#%E5%9B%9E%E8%BB%A2%E6%A5%95%E5%86%86%E4%BD%93%E9%9D%A2%E3%81%AB%E6%B2%BF%E3%81%86%E6%9C%80%E7%9F%AD%E8%B7%9D%E9%9B%A2%E3%81%AE%E5%BC%8F) にもある。

### (余談) 変数名などでどうやって省略するか

議論になりがちな話題として、「緯度(latitude) と経度(longitude) をどう略すか」というものがある。変数名として使うときはフルに `latitude` などと書くのではなく省略したいケースが多いが、どのように書くかはブレがある。

緯度 `latitude` の略は `lat` で確定と言える。4文字にするなら `lati` だけど、特に4文字にする意味がないのでほとんど使われない。

問題は経度 `longitude`。昔から `lng` 派と `lon` 派がずっと抗争を続けていて、どちらの派閥も自分たちの優位性を主張しているけど、統一には至っていない。3文字で決まらないのならば4文字の `long` にすればという意見もあるけど、`lat` との整合性で3文字にしたいところだし、`long` は一部のプログラミング言語で予約語になっていることがあるので使えないことがある。ということで3文字にするしかないけど、`lng` と `lon` が両立したまま今に至る。

そんなわけで、システムやサービス等ごとに `lng` と `lon` の違いがあるので注意が必要。とは言え現代では `lng` が優勢であり、MapLibre や Leaflet といった GIS ライブラリでは基本的に `lng` が使われていて、例外的に GPX のファイルフォーマットでは `lon` が使われている、程度に落ち着いてきている感じがある。

### その他

実際に座標を深堀りしていくと、測地系やジオイドなど、さまざまな知識が必要になってくる。このあたりは[国土地理院のページ](https://www.gsi.go.jp/sokuchikijun/datum-main.html)が詳しいので、こちらを参照。

## 磁北と真北

単純に「北」と言ったとき、2つの基準がある。

真北（しんぽく）は本当の北で、北極点の方向。GIS で「北」と言ったときにはこちらを指す。

磁北（じほく）は方位磁針のN極が指す方向。日本ではおよそ真北より西に7度程度ズレている（場所によって異なる）。空港の滑走路番号はこれに基づいて決められている。

## 緯度経度の GIS での表現

GIS で緯度経度を扱う場合、データとしての扱いやすさから、一般的によく使われる「度分秒」とは異なる形式で表現されることが多い。

### 度分秒

度を60進法で表すのがこの形式。**DMS形式**と呼ばれることもある。大雑把に言えば、世間でよく見る「東経139度44分28.8869秒　北緯35度39分29.1572秒」というやつ。

度(D)・分(M)・秒(S) の値で表し、分と秒が時刻と同じように60で繰り上がる。したがって 1度 = 60分 = 3600秒 となり、計算がとても面倒でコンピュータ向きではない。ただし航海における便利さなどの歴史的経緯により、コンピュータ以外の世間一般ではほとんどこれが使われている。

この形式のややこしさの一端が、何種類もの表現方法があること。以下に一部の例を示すが、各桁数の表記が変わることもあってややこしい。

- DD度MM分SS.SSSS秒
- DD°MM′SS.SSSS″
- DDMMSS.SSSS

### 度（10進度）

GIS で最も良く使われるのが「度」のみを使用する形式。10進度、十進角、Decimal degrees (DD) ともいう。

これは緯度経度の数値を**10進数の度**で表したもので、単純に数字が**度**を示す。そして東西南北を符号で表し、北緯・東経をプラス（符号なし）、南緯・西経をマイナスで表すことが多い。（まれに、東西南北のEWSNの1文字を書き加える方式も見かける）

この10進度はとても分かりやすく、単純な10進数の数値なので計算がとても簡単。そんなわけでコンピュータの世界ではよく使われる。

数値的には、およそ小数点以下6桁あれば実用的に充分な精度となる。0.000001度は、（場所と示す方向によるものの）大体10cmくらいの値になるので、GNSSの計測精度を考えるとこれくらいで充分。もう1桁追加して7桁にすれば完璧。

### 度分

度(D)と分(M)を `DDMM.MMMM` （度が3桁の場合は `DDDMM.MMMM`）と表記する方式。

「NMEA 0183」という規格でGNSSデバイスからシリアル通信で情報を取得するときの緯度経度がこの方式になっているため、GIS でも見かけることがある。

10進度へ変換する場合、分(M)の値を60で割ってから度に加えれば良い。小数点を基準にした文字列分割が必要になるが、比較的容易ではある。

### その他の形式

他にも、球体（曲面）ではなく平面として見る平面直角座標なんかもあったりしてここでは語り尽くせないので、興味がある人は調べてみると良さそう。

## MapLibre GL JS での座標の扱い

実際の地図表示ライブラリ MapLibre GL JS で座標をどのように扱うかを TypeScript の定義を元に見ていく。

座標は `LngLat` クラスとして定義されており、コンストラクタの定義は以下の通り。したがって `経度・緯度` の順に値（単位は 度）を指定する。**日本語の「緯度経度」とは逆**なので注意。

```ts
new LngLat(lng: number, lat: number): LngLat
```

ただし実際には `LngLatLike` を使用できるところがほとんどであるため、取り回しやすいこちらを使うことが多い。

```ts
LngLatLike = LngLat | { lat: number; lng: number; } | { lat: number; lon: number; } | [number, number]
```

単純な2つの数値のタプル（`経度・緯度` の順）だけでなく、オブジェクトも使えるのが便利。`lng` と `lon` の両方に対応しているのが懐の深さである。

`LngLatLike` の値は `LngLat.convert()` により `LngLat` へ変換できる。そして2つの `LngLat` の距離を `distanceTo()` により取得可能なので、難しい計算のことを任せられるので便利。

```ts
const tokyoStation = new LngLat(139.7669399, 35.6811558);
const tokyoSkyTree: [number, number] = [139.8106974, 35.7100349];
console.log(tokyoStation.distanceTo(LngLat.convert(tokyoSkyTree))); // → 5091.7876431319355
```

- [ドキュメント: LngLatLike](https://maplibre.org/maplibre-gl-js/docs/API/type-aliases/LngLatLike/)
